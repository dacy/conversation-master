# Handoff — ClipLingo prototype

Context for an agent (or person) picking this up on a local machine. The
prototype was built and validated in a Claude Code on the web sandbox whose
network policy blocks youtube.com / tiktok.com / npr.org, so **live fetching
was never run end-to-end** — that is the single most important thing left to
confirm locally.

## What this is

A prototype to validate one idea: can we automatically fetch real online media
(YouTube, TikTok, NPR) and turn it into short, quiz-able segments for a
TikTok-style scrolling language-learning feed? See `README.md` for the product
pitch and per-source feasibility table.

## Current state (branch `claude/language-learning-clips-prototype-oekt4s`, merged to `main`)

Everything below is committed and pushed.

- **Pipeline** (`pipeline/`, pure Python + yt-dlp): fetch → segment → generate
  questions → write `web/data/manifest.json`.
  - `sources/youtube.py` — yt-dlp search or URLs, downloads audio + captions.
  - `sources/tiktok.py` — yt-dlp, URL-only (no search on TikTok).
  - `sources/npr.py` — podcast RSS → direct mp3, no API key.
  - `transcript.py` — WebVTT parsing (dedupes yt-dlp rolling auto-subs).
  - `segment.py` — cue-grouping (10–35s, sentence boundaries) + ffmpeg
    silence fallback + clip cutting.
  - `questions.py` — offline cloze (fill-in-blank) MCQ from the transcript.
  - `manifest.py` — clip cutting orchestration + manifest assembly.
  - `demo.py` — offline espeak-ng synthesized episodes (network-free E2E test).
- **Player** (`web/`): dependency-free HTML/CSS/JS vertical snap-scroll feed;
  IntersectionObserver auto-plays the visible clip. Verified in Chromium
  (screenshots were shared in the web session).
- **Packaging**: `pyproject.toml` (hatchling), `uv.lock`, `cliplingo` console
  script. `requirements.txt` is just `-e .`.
- **Web session setup**: `.claude/hooks/session-start.sh` + `.claude/settings.json`
  install ffmpeg/espeak-ng/yt-dlp automatically in future web sessions. No-op
  locally (guarded by `CLAUDE_CODE_REMOTE`).

## Run it locally

```bash
# system deps (uv can't install these):
#   macOS:  brew install ffmpeg espeak-ng
#   ubuntu: sudo apt install ffmpeg espeak-ng
uv sync
uv run cliplingo demo            # offline, should always work → 9 clips
uv run cliplingo serve           # http://localhost:8000
```

## The immediate next task: confirm live fetching

This is where the user is headed. Run:

```bash
uv run cliplingo youtube "easy english conversation" --max 3
```

The fetcher now prints *why* it fails (recent fix, commit `ae84a62`). Expected
failure modes and fixes:

1. **All results skipped as too long** → output shows `skipped ... > 900s limit`.
   Fix: `--max-duration 3600`, or a shorter-content query.
2. **YouTube bot-check** ("Sign in to confirm you're not a bot") → add
   `--cookies-from-browser chrome` (or firefox/safari/edge).
3. **ffmpeg missing** → the command checks upfront and says so.
4. Use `--verbose` to see raw yt-dlp output.

Once a real fetch succeeds, verify the resulting clips + questions in the
player (`uv run cliplingo serve`), and update the feasibility table in
`README.md` with real results per source (esp. NPR needs a speech-to-text pass
before it can generate quizzes — currently listen-and-repeat only).

## Known limitations / roadmap

- **Question quality**: cloze-from-transcript is a placeholder. Swapping in an
  LLM call in `questions.py` is the obvious upgrade for real comprehension Qs.
- **NPR/TikTok have no transcripts** → no quizzes until a STT step (e.g.
  faster-whisper) is added; wire it in `manifest.py` before segmentation when
  `item.cues` is empty.
- **No automated tests** — `python -m compileall pipeline` is the only check;
  `cliplingo demo` is the de-facto E2E test.
- **Rights**: fine for a personal prototype; a real product needs licensed
  content / embeds / user-supplied URLs.
